# Telegram-бот для выдачи VST-плагинов и уведомлений о новых версиях

Это обновлённый каркас бота на `pyTelegramBotAPI`, который:

- выдаёт установщики VST-плагинов для Windows;
- перед выдачей просит пройти короткую проверку на человека;
- позволяет админу добавлять плагины в витрину;
- позволяет админу обновлять версию плагина без удаления карточки;
- позволяет заменять установщик или переключать плагин на внешнюю ссылку;
- умеет уведомлять пользователей, которые уже скачивали конкретный плагин, о выходе новой версии;
- пишет статистику пользователей, выдач и уведомлений в SQLite.

## Что внутри

- `bot.py` — основной код бота
- `requirements.txt` — зависимости Python
- `.env.example` — пример переменных окружения
- `plugins.example.json` — пример списка плагинов
- `plugins.json` — текущая витрина
- `bot.db` — база SQLite, создаётся автоматически при первом запуске

## Быстрый запуск

Скопируйте `.env.example` в `.env` и вставьте токен от `@BotFather`.

Проверьте `ADMIN_IDS` в `.env` и впишите свой Telegram user_id.

Установите зависимости:

```bash
pip install -r requirements.txt
```

Если на хостинге нет `python-dotenv`, бот всё равно запустится: он умеет читать `.env` сам, а при наличии переменных окружения возьмёт их напрямую из среды.

Запустите бота:

```bash
python bot.py
```

## Как теперь обновлять версию и установщик

Через `/admin` появилась кнопка `Обновить плагин`.

Дальше схема такая.

Сначала выбираете нужный плагин. После этого бот показывает карточку и три действия:

`Сменить версию` — меняет только поле версии в витрине.

`Заменить установщик` — позволяет загрузить новый файл или заменить старый установщик на внешнюю ссылку. Перед заменой бот спросит новую версию. Если ввести `-`, версия останется прежней.

`Уведомить скачавших` — бот отправит сообщение всем пользователям, которые раньше уже скачивали этот плагин. Для одной и той же версии повторная рассылка одним и тем же людям не дублируется.

## Как работает уведомление о новой версии

Бот смотрит таблицу `download_logs`, находит всех уникальных пользователей, которые ранее скачивали конкретный плагин, и рассылает им уведомление в личные сообщения.

В сообщении есть кнопка `Получить установщик`. Пользователь нажимает её и получает актуальную версию из витрины.

Важно понимать трезво: если человек заблокировал бота, удалил чат или никогда не запускал диалог после ограничений Telegram, часть уведомлений может не дойти. Это нормальное ограничение платформы, а не ошибка вашей логики.

## Как бот обращается с файлами

При загрузке установщика админом бот сохраняет `telegram_file_id`. Это важно, потому что при дальнейшей выдаче бот может пересылать уже сохранённый Telegram-файл без повторной тяжёлой заливки с вашего компьютера.

Если файл до 20 МБ, бот дополнительно скачивает его локально в папку `releases` и считает `SHA-256`.

Если файл больше 20 МБ, бот может сохранить только `file_id`. Это связано с ограничениями облачного Bot API Telegram на `getFile`.

## Команды

`/start` — приветствие и список плагинов

`/plugins` — список плагинов

`/help` — то же меню

`/admin` — админ-панель

`/stats` — статистика, только для `ADMIN_IDS`

`/cancel` — отмена текущего админ-действия

## Что лучше добавить следующим этапом

Следующий здравый шаг — сделать отдельное редактирование описания, changelog для каждой версии и кнопку `Что нового`, чтобы уведомление вело не просто на загрузку, а на короткое описание изменений. Если позже плагин станет платным, поверх этого уже можно спокойно надстроить генерацию лицензий и активацию.
