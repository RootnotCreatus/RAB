# Telegram-бот для бесплатной выдачи VST-плагинов под Windows

Это рабочий каркас бота на `pyTelegramBotAPI`, который показывает витрину плагинов, просит пройти короткую проверку на человека перед выдачей и ведёт SQLite-лог скачиваний.

Внутри есть административный режим. Администратор может открыть панель, добавить новый VST-плагин на витрину, загрузить к нему установщик прямо в Telegram как документ, либо указать внешнюю ссылку. Также можно удалить плагин с витрины без ручного редактирования `plugins.json`.

Теперь в проекте убрана прослойка выбора ОС. Логика упрощена: все плагины считаются сборками под Windows, поэтому при добавлении плагина администратор больше не выбирает платформу отдельно.

## Структура проекта

`bot.py` — основной код бота.

`requirements.txt` — зависимости Python.

`.env.example` — пример переменных окружения.

`plugins.example.json` — пример структуры файла с витриной.

`plugins.json` — реальная витрина, которую читает бот.

`bot.db` — база SQLite, создаётся автоматически при первом запуске.

`releases/` — папка, куда бот складывает установщики, загруженные администратором.

## Быстрый запуск

Скопируйте `.env.example` в `.env` и вставьте токен от `@BotFather`.

Скопируйте `plugins.example.json` в `plugins.json`, если хотите заранее заполнить витрину вручную. Если не хотите, можно просто запустить бота и добавлять плагины потом через `/admin`.

Установите зависимости:

```bash
pip install -r requirements.txt
```

Запустите бота:

```bash
python bot.py
```

## Как работает витрина

Для пользователя доступны команды `/start`, `/plugins` и `/help`. Бот показывает список доступных VST-плагинов. После нажатия на нужный плагин бот запускает короткую проверку на человека и затем отправляет файл или ссылку.

Если плагин хранится в Telegram-режиме, бот берёт путь из `plugins.json`, вычисляет SHA-256 и отправляет документ пользователю. Если плагин задан как ссылка, бот отправляет карточку с описанием и URL на скачивание.

Так как проект теперь ориентирован только на Windows, пользователю не предлагается выбирать платформу.

## Как работает панель администратора

Откройте `/admin` от аккаунта, который указан в `ADMIN_IDS`.

В панели доступны четыре действия: добавление плагина, удаление плагина, просмотр списка витрины и просмотр статистики.

### Добавление нового плагина

После выбора пункта «Добавить плагин» бот последовательно попросит:

slug,
название,
версию,
описание,
примечание к сборке,
тип выдачи.

Платформа отдельно не спрашивается. Для новых записей бот автоматически ставит `Windows`.

На последнем шаге можно выбрать один из двух вариантов.

Если выбрать «Загрузить файл», бот попросит отправить установщик документом. После получения файла он сохранит его в папку `releases/<slug>/`, посчитает SHA-256 и автоматически запишет новый плагин в `plugins.json`.

Если выбрать «Указать ссылку», бот попросит URL и добавит плагин на витрину как внешний релиз.

### Удаление плагина

При удалении бот убирает запись из `plugins.json`. Если файл ранее был загружен через административную панель и лежит внутри папки `releases/`, бот попытается удалить и сам установщик.

### Отмена текущего действия

Если администратор начал добавление плагина и передумал, можно отправить `/cancel`.

## Команды

Пользовательские команды:

`/start`
`/plugins`
`/help`

Административные команды:

`/admin`
`/stats`
`/cancel`

## Что важно учесть на практике

Бот хранит витрину в `plugins.json`, а не в отдельной таблице базы. Это сделано специально, чтобы список релизов оставался прозрачным и его можно было при желании править руками.

Файлы, отправленные администратором, сохраняются на диске. Поэтому для постоянной работы лучше держать бота на VPS или сервере с постоянным хранилищем, а не на домашнем ПК.

Если один и тот же файл уже был отправлен пользователю через Telegram, бот кэширует `file_id` и затем может раздавать этот установщик быстрее.
